# Audio Implementation Notes

This document captures research and recommendations for implementing cross-platform audio in Pandowdy.

## Requirements

For Apple IIe emulation, we need:
- **Low latency** (< 10ms buffer) - critical for speaker clicks
- **Simple PCM output** - mono, 44.1kHz or 48kHz
- **Real-time waveform generation** - Apple IIe uses 1-bit audio (speaker toggles)
- **Cross-platform** - Windows, Linux, macOS
- **Lightweight** - avoid massive dependencies like SDL if possible

## Apple IIe Audio Characteristics

The Apple IIe has a very simple audio system:
- **1-bit speaker** - Just toggles between two states (click!)
- **CPU-controlled** - Software toggles memory address $C030 to click the speaker
- **No dedicated audio hardware** - All audio is synthesized by precisely-timed CPU operations
- **Typical usage** - Beeps, clicks, and simple waveforms generated by toggling at specific frequencies

### 🏆 **Recommended: OpenAL Soft via Silk.NET.OpenAL**

**Package:**

- Won't bloat the application

**Deployment:**
- Include `soft_oal.dll` in Windows builds
- Linux/Mac: System OpenAL usually sufficient
- Can use NuGet native package helpers for bundling

## Implementation Sketch (Silk.NET.OpenAL)

### Basic Setup

```csharp
using Silk.NET.OpenAL;

public class AppleIIAudio : IDisposable
{
    private AL _al;
    private ALContext _alc;
    private Device* _device;
    private Context* _context;
    private uint _source;
    private Queue<uint> _bufferPool;
    private short _currentAmplitude;
    
    public const int SampleRate = 44100;
    public const int BufferSize = 882; // ~20ms @ 44.1kHz (50 Hz frame rate)
    
    public void Initialize()
    {
        _al = AL.GetApi();
        _alc = ALContext.GetApi();
        
        // Open default audio device
        _device = _alc.OpenDevice(null);
        if (_device == null)
            throw new Exception("Failed to open audio device");
        
        // Create audio context
        _context = _alc.CreateContext(_device, null);
        _alc.MakeContextCurrent(_context);
        
        // Create source for playback
        _source = _al.GenSource();
        _al.SetSourceProperty(_source, SourceFloat.Gain, 0.5f);
        
        // Pre-allocate buffer pool
        _bufferPool = new Queue<uint>();
        for (int i = 0; i < 4; i++)
        {
            _bufferPool.Enqueue(_al.GenBuffer());
        }
    }
    
    public void OnSpeakerToggle(bool state)
    {
        // Apple IIe speaker: toggle between +/- amplitude
        _currentAmplitude = state ? short.MaxValue : short.MinValue;
    }
    
    public void SubmitFrame(short[] pcmData)
    {
        // Unqueue processed buffers and return to pool
        _al.GetSourceProperty(_source, GetSourceInteger.BuffersProcessed, out int processed);
        for (int i = 0; i < processed; i++)
        {
            uint buffer = _al.SourceUnqueueBuffers(_source, 1)[0];
            _bufferPool.Enqueue(buffer);
        }
        
        // Get a free buffer from pool
        if (_bufferPool.Count == 0)
            return; // All buffers in use, skip this frame
        
        uint nextBuffer = _bufferPool.Dequeue();
        
        // Upload PCM data
        unsafe
        {
            fixed (short* ptr = pcmData)
            {
                _al.BufferData(nextBuffer, BufferFormat.Mono16, ptr, 
                    pcmData.Length * sizeof(short), SampleRate);
            }
        }
        
        // Queue buffer
        _al.SourceQueueBuffers(_source, 1, new[] { nextBuffer });
        
        // Start playing if stopped
        _al.GetSourceProperty(_source, GetSourceInteger.SourceState, out int state);
        if (state != (int)SourceState.Playing)
            _al.SourcePlay(_source);
    }
    
    public void Dispose()
    {
        if (_al != null)
        {
            _al.DeleteSource(_source);
            foreach (var buffer in _bufferPool)
                _al.DeleteBuffer(buffer);
        }
        
        if (_alc != null)
        {
            _alc.DestroyContext(_context);
            _alc.CloseDevice(_device);
        }
    }
}
```

### Integration with VA2M Emulator

```csharp
// In VA2M.cs or new AudioAdapter.cs
public class AudioAdapter
{
    private AppleIIAudio _audio;
    private short[] _frameBuffer;
    private int _samplesPerFrame;
    private bool _speakerState;
    
    public AudioAdapter(int frameRate = 60)
    {
        _audio = new AppleIIAudio();
        _audio.Initialize();
        
        // Calculate samples per frame (e.g., 44100 / 60 = 735 samples)
        _samplesPerFrame = AppleIIAudio.SampleRate / frameRate;
        _frameBuffer = new short[_samplesPerFrame];
    }
    
    public void OnSpeakerAccess()
    {
        // Apple IIe: Accessing $C030 toggles speaker
        _speakerState = !_speakerState;
        _audio.OnSpeakerToggle(_speakerState);
    }
    
    public void OnFrameComplete()
    {
        // Fill frame buffer with current speaker state
        short amplitude = _speakerState ? short.MaxValue : short.MinValue;
        for (int i = 0; i < _frameBuffer.Length; i++)
        {
            _frameBuffer[i] = amplitude;
        }
        
        // Submit to audio system
        _audio.SubmitFrame(_frameBuffer);
    }
}
```

### Soft Switch Handler

```csharp
// In VA2MBus.cs or SoftSwitchHandler
public byte ReadSpeaker(ushort address)
{
    if (address == 0xC030)
    {
        _audioAdapter?.OnSpeakerAccess();
    }
    
    return 0xFF; // Floating bus value
}

public void WriteSpeaker(ushort address, byte value)
{
    if (address == 0xC030)
    {
        _audioAdapter?.OnSpeakerAccess();
    }
}
```

## Deployment Considerations

### Windows
- Bundle `soft_oal.dll` (OpenAL Soft) in application directory
- Can use NuGet package: `OpenAL.redist` or manual download
- Size: ~500KB

### Linux
- OpenAL Soft usually pre-installed via `libopenal1` package
- If not: User installs `libopenal-dev` or `openal-soft`
- No bundling needed

### macOS
- OpenAL framework built into macOS
- No bundling needed
- Works out of the box

## Future Enhancements

Once basic audio is working, consider:

### Advanced Waveform Generation
- **Interpolation** - Smooth transitions between speaker states
- **Band-limiting** - Anti-aliasing for cleaner high-frequency tones
- **Filtering** - Low-pass filter to emulate Apple IIe speaker characteristics

### Audio Settings
- **Volume control** - User-adjustable gain
- **Mute option** - Disable audio entirely
- **Latency adjustment** - User can tune buffer size vs. latency tradeoff

### Debugging Tools
- **Waveform visualizer** - Show audio output in real-time
- **Speaker toggle counter** - Debug timing issues
- **Audio buffer stats** - Monitor underruns/overruns
